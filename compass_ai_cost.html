Compass AI - Cost Analysis
How the AI works inside Compass, what it costs at every scale, and which model to use. Based on one Compass completion per user per month (reassessment frequency). Stage Score now AI-calculated.
How Compass AI Works
1
Adaptive Survey Branching
Founder answers a question. AI reads the answer and decides which question comes next - skipping irrelevant sections, digging deeper where needed. Runs on 5–8 key questions (the free-text and nuanced ones). Simple yes/no branches are rules-based, no AI.
AI: 5–8 API calls per survey
2
Stage Score Calculation
Once survey is complete, AI calculates the Stage Score (0–1000) across 5 dimensions - business model, team, market, founder, and execution. AI weighs each answer contextually, not just numerically, producing a more nuanced score than a fixed formula. One API call.
AI: 1 API call after survey
3
Founder Profile Generation
AI reads all survey answers and generates a structured founder profile - stage, archetype, top 3 challenges, goals, capital readiness. This is an extraction task. One API call. The profile drives every personalized feature downstream.
AI: 1 API call at end of survey
4
Personalized Path Generation
AI takes the founder profile and maps it to the right milestone template for their stage. Prioritizes steps based on challenges. Returns 4–6 ordered milestones with matched lesson IDs. One API call.
AI: 1 API call at end of survey
5
Partner Trigger Detection
AI scans all free-text answers for business signals - cashflow stress, exit intent, fundraising timeline, hiring pressure. Each signal maps to a partner category (CFO, M&A, legal, etc.). Fires trigger events in the database. One API call.
AI: 1 API call at end of survey
6
Advisor Qualification
AI matches the founder profile to available advisor specialties. Scores relevance between founder's stage/challenges and each advisor's focus area. Returns ranked advisor matches for the dashboard. One API call.
AI: 1 API call at end of survey

Model Recommendation
✦ Use GPT-4o mini for Compass. Compass does not need a top model.
Why Compass doesn't need a top model
Every AI task in Compass is structured. Inputs are survey answers (mostly multiple choice, short text). Outputs are decisions with pre-defined options (next question, profile fields, milestone templates, trigger categories). This is classification and extraction - not open-ended reasoning. Small, fast models handle this perfectly.
Why GPT-4o mini specifically
Best reliability-to-cost ratio in the market. Widely used in production, well-documented, stable API. 22× cheaper than Claude Sonnet for identical Compass tasks. Handles structured JSON output reliably - critical for the profile and path generation steps. Easy to swap to GPT-4o only for edge cases if needed.
What top models are actually for
Atlas AI - where founders ask complex, open-ended business questions and need nuanced, accurate advice. Domain specialists (CFO, Legal, M&A) where reasoning quality matters. Using Claude Sonnet or GPT-4o for Compass is like hiring a senior consultant to fill out a form.
Cost impact of the wrong choice
At 1,000 users: GPT-4o mini costs $54/year. Claude Sonnet costs $1,224/year - 22× more. At 100,000 users: GPT-4o mini = $5,400/year. Claude Sonnet = $122,400/year. The wrong model choice at scale costs $117,000/year in unnecessary AI spend.
New Users vs Existing Users
New User
One-time cost on signup
User signs up → completes Compass for the first time → all 6 AI tasks run once. Profile created, Score calculated, Path generated, Triggers detected, Advisors matched.
Cost per new member
$0.006
Fixed cost per acquisition. Happens once regardless of how long the member stays.
Existing User
Recurring every 30 days
Compass reassesses every 30 days to track progress. Each reassessment runs the same 6 AI tasks - updated Score, refreshed Path, new trigger detection, re-qualified advisors.
Cost per active user/month
$0.006
Inactive users cost $0 - no reassessment triggered, no AI call made.
